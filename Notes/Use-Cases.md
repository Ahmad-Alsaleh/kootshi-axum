# Use Cases

## Milestone 1: basic use cases (bare minimum for a MVP)

- user creates a company account
- user creates a player account
- user creates a new pitch
    - user needs to be logged in using a company account
- user adds availability time slots to a pitch
    - user needs to be logged in using a company account
- user views available pitches
- user views availability time slots of a specific pitch
- user books a time slot of a specific pitch
    - user needs to be logged in using a player account

## Milestone 2: better CRUD

- user cancels a booking
    - user needs to be logged in using a player account
- user modifies pitch info
    - user needs to be logged in using a company account
- user deletes availability time slot of a specific pitch
    - user needs to be logged in using a company account

## Milestone 3: payments

> user needs to be logged in using a player account for all of the bellow

- user pays the full amount in one installement
- user pays a single portion
- user generates a payment link to share with other players
    - other players do not need to be logged in to pay
- user pays the full amount in two installements (one advance installement)
